Tab Manager AI is designed with security and privacy as our top priorities. This Privacy Policy describes what data the extension collects, how it is used, and how it is protected. Your collections and settings are stored locally on your device. The only external communication occurs when you use the optional AI tab grouping feature, which sends tab titles and URLs to an AI provider for processing.

01 — DataData Collection & Usage

1.1 Data stored locally

The following is stored locally on your device using Chrome's secure storage APIs and is never transmitted to any server:

  • Tab collections (names, tab titles, URLs, favicons)
  • User settings and preferences
  • Session history
  • Undo/redo action history
  • AI usage counters (for rate limiting)

1.2 Data shared with third parties

When you use the AI Group feature, the following is sent to our AI provider over an encrypted HTTPS connection:

  • Tab titles
  • Tab URLs
  • Your grouping instruction (e.g., "group by project")

This data is used solely to generate grouping suggestions. No other personal data is transmitted. The AI grouping feature is optional and limited to 3 uses per day on the Free plan.

What we never send Browsing history, passwords, cookies, personal information, analytics, advertising identifiers, or telemetry of any kind.

1.3 Data NOT collected

  • No browsing history
  • No passwords or credentials
  • No personal information (name, email, etc.)
  • No cookies or tracking data
  • No telemetry or analytics data
  • No advertising identifiers

02 — PermissionsWhy each permission exists

  • tabs — read open tab titles and URLs to display and organize them
  • storage — store collections, settings, and session history locally
  • alarms — schedule auto-save and backup reminders
  • contextMenus — add right-click menu options for quick tab saving
  • notifications — send backup reminders when tabs haven't been saved (disable-able)

Host permissions

The extension communicates with our AI provider's API only when you invoke the AI Group feature. All communication uses HTTPS encryption.

03 — SecuritySecurity Architecture

Local-first design

  • All collections and settings stored locally on your device
  • No user accounts or authentication required
  • No cloud storage or remote database
  • Extension works fully offline (except AI grouping)

Data isolation

  • Extension data isolated from other browser data
  • No access to browsing history, passwords, or personal information
  • Sandboxed execution environment
  • No cross-extension data sharing

Data encryption & code security

  • All stored data uses Chrome's built-in encryption
  • All network communication uses HTTPS encryption
  • All extension code is static and locally stored — no dynamic code loading
  • Regular security reviews of source code
  • Input validation and sanitization throughout

04 — ControlYour Rights

  • Complete control over your data at all times
  • Export your collections in JSON, CSV, Markdown, or TXT
  • Delete any or all data at any time
  • AI grouping is entirely optional — the extension works fully without it
  • Uninstalling the extension removes all locally stored data

Data retention

  • Data is retained only as long as the extension is installed
  • No data is retained on external servers after AI processing
  • Session history has automatic cleanup

05 — ComplianceRegulatory & Standards

  • GDPR compliance for EU users
  • CCPA compliance for California users
  • Chrome Web Store Developer Program Policies
  • OWASP security guidelines
  • Industry security best practices
