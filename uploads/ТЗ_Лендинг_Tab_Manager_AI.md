## ЧАСТЬ 1. АНАЛИЗ КОНКУРЕНТА

> Источник: все три страницы принадлежат одному продукту — **ATO (atobase.app)**. Это главная страница, страница Tab Observer (feature-page) и страница Pricing.

### Структура, извлечённая из конкурента

| Страница | Блоки |
|---|---|
| Главная | Nav → Hero → Features (сетка) → Feature Deep Dive (секции) → CTA-баннер → Footer |
| Feature Page (Tab Observer) | Nav → Hero с видео → Intro-секция → Feature Sections (поочерёдно) → Grid из 4 фич → Footer |
| Pricing | Nav → Заголовок → Промо-баннер (скидка + купон) → Карточки тарифов (Free / Pro) → FAQ → Footer |

### Повторяющиеся паттерны

- Навбар: логотип + ссылки на Features / Feature Page / Pricing / Contact + кнопка Sign In (или Get Extension)
- Hero с градиентным заголовком (purple-to-indigo) и подзаголовком
- Фичи через комбинацию: большой иконический текст слева + описание справа (чередование сторон)
- CTA-секция с тёмным фиолетовым фоном и кнопкой на Chrome Web Store
- 2-колоночный pricing (Free + Pro), Pro с бейджем "7 days free trial" и градиентной рамкой
- Промо-баннер с кодом скидки прямо над карточками тарифов
- Footer: копирайт + Terms / Privacy / Refund

### Логика структуры

1. Захват внимания (Hero) → 2. Демонстрация ценности (Features) → 3. Углублённое объяснение ключевой фичи (Feature Deep Dive) → 4. Социальное доказательство / доверие → 5. Конвертация (Pricing) → 6. Финальный призыв к действию (CTA) → 7. Footer

**Рекомендации по визуалу:** Sticky-навбар с лёгким blur/backdrop при скролле. Цветовая схема: синий/бирюзовый или сине-фиолетовый (НЕ копировать purple конкурента — взять свой акцент).

### БЛОК 2: HERO

**Цель:** Захватить внимание, сформулировать главную ценность за 3 секунды, дать CTA.

**Содержимое обязательно:**
- H1: Главный заголовок — проблема + решение в одной фразе
- Подзаголовок: 1–2 предложения, раскрывают механизм (AI + ручное управление)
- Первичная CTA-кнопка: "Add to Chrome — Free" (с иконкой Chrome)
- Вторичная ссылка: "See how it works ↓"
- Визуальный элемент: скриншот/мокап интерфейса расширения (панель с коллекциями и AI-кнопкой)

**Данные из продукта:**
- Проблема: tab overload, хаос вкладок
- Решение: AI-группировка одним кликом + коллекции + история
- Подтверждение: GPT-4o-mini, без регистрации, локальное хранение

**Рекомендации по тексту:**
- H1 примерный смысл (НЕ копировать дословно): "Ваши 50 вкладок — организованы за секунду"
- Акцент на: "один клик", "без регистрации", "бесплатно"
- Тон: уверенный, конкретный, без маркетинговых пустышек типа "revolutionary"

**Рекомендации по визуалу:**
- Мокап расширения с видимой кнопкой "AI Group" и деревом коллекций
- Фон: мягкий градиент или сетка (mesh gradient), не белый
- Анимация: fade-in с лёгким смещением

## ЧАСТЬ 4. ГЛОБАЛЬНЫЕ ИНСТРУКЦИИ ДЛЯ ИИ-ГЕНЕРАТОРА

### Стиль лендинга
- **Уровень продаваемости:** умеренный (не агрессивный, не скучный)
- **Тон голоса:** дружелюбный эксперт. Говорит с пользователем как равный, объясняет конкретно
- **Язык:** английский (основной). Допустима локализация под целевой рынок
- **Длина текстов:** заголовки до 10 слов, подзаголовки до 25, описания в карточках до 40

### Визуальный стиль
- Цветовая схема: выбрать НЕ purple — например slate-blue + teal accent, или navy + amber
- Типографика: выразительный display-шрифт для заголовков + читаемый sans-serif для body
- Фон: не белый — мягкий mesh gradient или subtle noise texture
- Анимации: staggered fade-in при загрузке, hover на карточках, плавный scroll

### Запреты (чего нельзя делать)

❌ Не писать "revolutionary", "game-changing", "next-level" — пустые слова  
❌ Не копировать тексты с сайта ATO (конкурент)  
❌ Не придумывать функции, которых нет в описании продукта  
❌ Не использовать stock-иллюстрации людей за ноутбуком — только UI-мокапы  
❌ Не ставить более 2 CTA-кнопок на один экран  
❌ Не делать Pricing без FAQ — это снижает конверсию  
❌ Не использовать Inter + purple gradient — слишком похоже на конкурента  
❌ Не добавлять блок testimonials если нет реальных отзывов (можно заменить на trust-факты)  

### Требования к уникальности
- Все тексты написать заново на основе описания продукта
- Структура взята у конкурента, контент — только из Tab Manager AI
- Визуальный язык должен отличаться от ATO (другой акцентный цвет, другая типографика)

## ЧАСТЬ 6. САМОКРИТИКА И СЛАБЫЕ МЕСТА ТЗ

### Идентифицированные риски и как их снизить:

**1. Нет реальных цифр пользователей**  
→ Заменить на trust-факты. Не выдумывать цифры. Если есть данные CWS (рейтинг, кол-во установок) — добавить в ТЗ перед генерацией.

**2. Нет скриншотов/мокапов**  
→ ИИ должен сгенерировать placeholder-блоки с описанием что должно быть на изображении, или использовать CSS-мокап интерфейса.

**3. Ценообразование взято у конкурента**  
→ Помечено как предположение. Владелец продукта должен подтвердить цены перед финальной генерацией.

**4. Нет данных о промокоде**  
→ Оставить `[PROMO_CODE]` как placeholder. Промо-блок сделать условным (если есть промокод — показать, если нет — убрать).

**5. Блок Social Proof слаб**  
→ До запуска использовать product-факты. После первых отзывов — добавить real testimonials и обновить лендинг.

## ЧАСТЬ 7. МУЛЬТИПЕРСПЕКТИВНАЯ ПРОВЕРКА

### Взгляд разработчика
- ТЗ описывает блоки, порядок и содержимое — реализация возможна в HTML/CSS или любом фреймворке
- Нет ссылок на конкретные компоненты — разработчик выбирает стек самостоятельно
- Pricing с monthly/yearly toggle — требует минимального JS (переключение цен)

### Взгляд ИИ-генератора
